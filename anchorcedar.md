# ChanCup Analytics Hub

ChanCup Analytics Hub 是一个面向茶叶竞技赛事的数据聚合与统计分析平台，专门用于采集、整理和可视化各类茶艺竞赛的实时比分、赛季积分榜以及历史赛事结果。项目定位为茶文化数字化基础设施，为赛事组织者、参赛选手、茶文化研究机构及爱好者提供权威、可追溯的赛事数据服务。

本项目通过分布式数据采集框架，整合多源异构的赛事公开数据，统一清洗后提供结构化查询接口与前端看板。目前系统已覆盖全国主要茶类竞技赛事的核心数据流，支持自定义时间段筛选、选手纵向追踪、区域横向对比等分析功能。项目采用模块化设计，核心数据处理层与展示层解耦，既可独立部署为后端 API 服务，也可无缝集成至现有茶文化门户网站或移动端应用。

## 功能概览

- **实时比分看板**：提供赛事进行中的动态分数更新面板，数据刷新间隔可配置，支持 WebSocket 推送与轮询双模式。

- **赛季积分榜生成**：根据赛事权重系数与选手名次自动计算赛季累积积分，支持多赛季并行管理与历史赛季回溯。

- **历史赛事结果检索**：按赛事名称、日期范围、茶类品种、举办地区等多维度组合条件检索已完成赛事的完整结果记录。

- **选手数据纵向分析**：针对单一选手生成跨赛季的积分走势曲线图、参赛频次统计及最佳成绩标注。

- **区域竞争力指数**：基于行政区划维度聚合赛事数据，输出区域间综合实力对比雷达图与排名变动趋势。

- **数据导出与报告生成**：支持将筛选后的数据集导出为 CSV 或 JSON 格式，并可自动生成 PDF 格式的赛事分析简报。

- **异常数据自动标记**：内建数据校验规则引擎，对异常分数、重复记录、逻辑矛盾条目进行自动标记并推送审核预警。

## 应用场景

- **赛事组织方赛后复盘**：赛事承办单位可在比赛结束后批量导入裁判评分表，系统自动生成结构化的赛事结果汇总页，并同步更新该赛事的历年对比数据，辅助组织方识别评分一致性与赛制优化空间。

- **茶文化研究机构趋势分析**：研究机构通过查询特定茶类（如绿茶、乌龙茶）近五年的赛事积分变动，结合时间序列模型，分析不同品种茶在竞技评价体系中的受关注度演变规律。

- **参赛选手个人数据追踪**：注册选手可绑定个人身份标识，系统自动聚合其参与过的所有场次得分与排名，生成个人职业生涯数据看板，用于备赛策略调整与能力短板识别。

- **媒体与内容创作者资讯采编**：茶文化记者或自媒体运营者利用公开赛事结果数据，快速获取某一时段内的热点选手与高关注度赛事信息，支撑新闻报道或专题内容的素材采集。

## 快速开始

以下步骤将指导您在本地环境中完成项目的克隆、依赖安装与开发服务启动。

```bash
# 1. 克隆代码仓库
git clone https://github.com/chan-cup/analytics-hub.git
cd analytics-hub

# 2. 安装项目依赖（使用 npm）
npm install

# 3. 配置环境变量（复制示例配置并修改数据库连接参数）
cp .env.example .env

# 4. 执行数据库迁移与种子数据填充
npx prisma migrate deploy
npx prisma db seed

# 5. 启动开发服务器（默认监听端口 3000）
npm run dev
```

访问本地服务地址后，系统将自动跳转至初始化配置向导页面，您可按照指引完成管理员账户创建与数据源连接测试。

## 常见问题

**问：系统支持导入哪些格式的历史赛事数据？批量导入时有哪些限制？**

答：当前版本支持 CSV 与 JSON 两种导入格式。CSV 文件需包含 `event_name`、`event_date`、`player_name`、`score`、`rank` 等必需列，编码建议使用 UTF-8。JSON 文件需符合系统提供的 Schema 定义。单次批量导入记录上限为 5000 条，超出部分建议分割为多个文件分批操作。导入过程中系统会进行实时校验，并在界面反馈错误行号与原因。

**问：如何在不同网络环境（内网、公网、离线环境）下部署本项目？**

答：项目通过环境变量 `NODE_ENV` 与 `DEPLOY_ENV` 组合控制运行模式。公网部署需正确配置 Nginx TLS 证书与域名解析；内网部署可关闭强制 HTTPS 并调整 CORS 白名单；完全离线环境需使用 `npm pack` 将依赖打包至内网仓库，并预先下载所有外部字体与图表库资源文件，部署时通过 `ASSET_BASE_URL` 指向本地静态资源服务器。

**问：积分榜的权重计算逻辑是否可以自定义？如何调整不同赛事类别的影响力系数？**

答：权重计算引擎支持动态配置。您可通过管理员后台的“赛事模板管理”功能，为每一类赛事（如国家级、省级、邀请赛）设定基础权重系数（范围 0.1 至 1.0）。系统默认算法为：赛季总积分 = 单场得分 × 赛事权重 × 难度修正系数。难度修正系数自动根据当年参赛人数与历史冠军平均分计算得出，您也可以在该页面手动覆盖该修正值。
