# TechLink Navigator

TechLink Navigator 是一个面向技术调研人员、开发者与开源项目维护者的结构化外链资源汇总与导航工具。该项目不存储任何实际内容，仅以目录化、分类化的方式聚合高质量技术参考资料，帮助用户在海量信息中快速定位所需的技术文档、数据源与社区入口。项目本身采用纯静态 Markdown 驱动，配合自动化索引生成工具，可实现每日资源 freshness 检查与失效链接告警，适用于个人技术收藏管理、团队知识库外链整合以及开源项目 README 中的参考资料编排。

该项目并非通用的书签管理器，而是专注于技术场景下的外链结构化组织。目标用户包括需要频繁查阅技术规范、竞赛数据、编程示例库的开发者，以及需要维护项目外部依赖文档列表的技术作者。通过明确的分类标签、更新时间戳与简短注释，TechLink Navigator 将原始 URL 转化为可维护、可审计、可共享的技术资源清单，降低信息检索的认知负担。

## 功能概览

- **自动分类索引**：根据 URL 域名与路径特征，自动将链接归入技术竞赛、编程示例、数据榜单或官方文档等预设类别，减少人工整理成本。
- **链接健康状态检测**：每日定时发起 HEAD 请求，标记返回非 200 状态码或超时的链接，并在索引报告中高亮显示失效项。
- **Markdown 原生渲染**：所有资源列表以纯 Markdown 表格或列表形式输出，兼容 GitHub、GitLab 及多数静态站点生成器，无需额外解析器。
- **自定义标签与注释**：支持为每条外链添加最多三个标签（如 "python", "benchmark", "official"）和一段 140 字以内的用途说明，便于后续过滤与搜索。
- **版本化快照导出**：每次更新时自动生成带时间戳的资源清单快照，允许回溯历史版本，便于审计外部依赖变更。
- **多格式输出支持**：除 Markdown 外，可通过插件输出 JSON、YAML 或 HTML 格式，满足不同文档系统的嵌入需求。
- **CLI 交互工具**：提供命令行交互模式，支持批量添加、删除、修改链接属性，并支持从 CSV 文件批量导入。
- **Webhook 通知集成**：当检测到大量链接失效（超过总数的 10%）时，可通过钉钉、飞书或 Slack Webhook 发送告警消息。

## 应用场景

- **技术调研阶段的资料整理**：调研人员在研究新型编程框架或算法竞赛方案时，可将分散在数十个标签页中的示例代码库、官方文档、性能榜单统一收录至 TechLink Navigator，并按主题标记，后续撰写技术报告时可直接引用索引条目。
- **开源项目 README 的外部依赖维护**：开源项目维护者需要列出项目所参考的第三方数据集、API 文档或基准测试网站。使用本工具可生成结构化的外链附录，并定期检查这些外部资源是否仍可访问，避免 README 中的链接逐渐失效。
- **团队知识库的公共资源入口管理**：技术团队内部 Wiki 常包含大量指向外部工具、云服务控制台、监控面板的链接。TechLink Navigator 可作为这些链接的中间管理层，统一添加访问频率、维护人、最后验证时间等元数据，减少成员因链接失效而浪费的排错时间。
- **编程竞赛或考试备考资料聚合**：针对特定技术认证或算法竞赛，考生需要同时参考多个评分系统、样题库和官方规则文档。本工具可将这些资源按考试科目或难度级别分层组织，形成清晰的备考导航页。
- **技术文档站点的外链附录生成**：技术写作者在编写手册或教程时，常在末尾附上参考链接。使用本工具可以自动化生成格式统一的附录，并支持按章节动态筛选相关链接，避免手动维护时格式混乱或链接重复。

## 快速开始

以下步骤将在本地环境中获取 TechLink Navigator 源码、安装依赖并启动基础索引服务。

```bash
# 克隆项目仓库
git clone https://github.com/techlink-navigator/tln-core.git
cd tln-core

# 安装依赖（使用 Python 3.9+ 及 pip）
pip install -r requirements.txt

# 执行初始资源索引生成（基于内置示例链接）
python cli.py build --input ./data/urls.csv --output ./dist/README.md

# 启动本地预览服务（默认端口 8080）
python cli.py serve --port 8080 --dir ./dist
```

## 常见问题

**Q: 资源列表中的链接访问时返回 404，工具会自动处理吗？**

A: 是的。CLI 的 check 子命令会每日检测所有链接的 HTTP 状态码。对于返回 404 或超时的链接，工具会在生成的索引报告中用 [失效] 标记，并记录首次检测到失效的时间。但工具不会自动删除或修改原始 URL 数据，需要用户根据报告手动确认后通过 edit 命令移除或更新。

**Q: 我可以使用自己的分类规则而不使用内置的域名分类器吗？**

A: 可以。您可以在配置文件中将 classifier.mode 设置为 custom，然后在 data/rules.yaml 中编写基于正则表达式或路径前缀的规则。规则文件支持优先级排序，并且可以针对特定 URL 完全覆盖默认分类。详细语法参考 docs/configuration.md 中的“自定义分类规则”章节。

**Q: 如何迁移现有的书签或浏览器收藏夹到本工具中？**

A: 项目提供了转换脚本 tools/import_browser.py，支持从 Chrome 或 Firefox 导出的 HTML 书签文件直接转换。转换时会自动提取书签名称作为注释，并尝试根据 URL 生成初始标签。若需要从 CSV 或通用 JSON 导入，可使用 cli.py import --format csv 命令，格式说明见 data/example_import.csv 示例文件。
