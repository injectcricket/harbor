# OpenLink Navigator

OpenLink Navigator 是一个面向技术调研与数字资源治理的开源外链聚合导航系统。该项目定位于为开发者、技术决策者及研究型用户提供高质量外部链接的集中式索引与分类管理能力，通过结构化数据组织与轻量级前端交互，降低信息过载环境下的资源发现成本。项目本身不生产内容，而是作为可信第三方的链接枢纽，解决个人书签分散、团队知识库链接失效、以及开源文档中外部引用混乱三类核心问题。

## 功能概览

- **多级分类索引**：支持按领域、应用场景、数据来源三个维度对链接进行动态标签化分组，便于快速定位相关资源。
- **链接状态巡检**：内置周期性可用性检测机制，自动标记响应超时或状态码异常的条目，辅助维护链接健康度。
- **原始数据透传**：严格保留用户提交的每一份原始 URL 字符串，不进行协议补全、域名规范化或路径重写，确保数据完整性。
- **静态站点生成**：基于配置文件一键生成纯静态 HTML 目录页，可直接部署至任意 Web 服务器或 CDN，无需运行时数据库。
- **Markdown 元数据驱动**：所有链接条目以 Markdown 列表形式存储于版本控制系统中，支持 PR 审核与变更追溯。
- **模糊搜索与过滤**：提供基于标题、标签、域名片段的前端快速过滤，支持多关键字组合查询。
- **导入导出兼容**：支持 CSV 与 JSON 格式的链接批量导入导出，便于与其他书签管理工具或爬虫数据源对接。

## 应用场景

- **开源项目外部依赖索引**：当开源项目需要引用大量第三方文档、数据源或社区论坛时，可使用本系统统一管理这些外部链接，避免在 README 或 Wiki 中罗列过长且难以维护的 URL 列表。
- **技术团队内部知识导航**：企业研发团队可将本系统部署于内部网络，集中收录云服务控制台、监控面板、内部 API 文档、CI/CD 流水线入口等日常高频链接，减少成员记忆负担。
- **数据采集任务资源登记**：数据工程师在构建多源爬虫或 ETL 任务时，可利用本系统登记目标数据源入口，配合状态巡检功能监控源站可用性变化，提前发现采集异常。
- **技术调研成果归档**：架构师或技术经理在进行竞品分析或新技术预研时，可将收集到的官方文档、性能测试报告、社区讨论帖等链接按项目维度归档，形成可复用的知识库。

## 快速开始

以下命令演示了从代码仓库克隆项目到本地运行开发服务器的完整流程。

```bash
# 克隆项目仓库
git clone https://github.com/openlink-navigator/navigator-core.git
cd navigator-core

# 安装项目依赖（使用 npm）
npm install

# 启动本地开发服务器（默认监听 3000 端口）
npm run dev
```

启动后，打开浏览器访问 `http://localhost:3000` 即可查看导航主页。所有外部链接数据存放于 `data/links.json` 文件中，可直接编辑该文件或通过管理界面进行增删改操作。

## 常见问题

**问：项目是否会主动修改或补全用户提交的 URL 字符串？**

答：不会。本项目严格遵循透传原则，所有 URL 在录入、存储、展示及导出过程中均保持原始字符串不变。系统不会自动添加 `http://` 或 `https://` 前缀，也不会移除 `www` 子域名或进行大小写转换。用户需要自行确保所提交 URL 的可访问性。

**问：内置的链接状态巡检功能是否会影响目标站点的正常访问？**

答：巡检功能仅发送标准的 HTTP HEAD 请求以检测响应状态码，不下载页面内容，不携带自定义 User-Agent 以外的特殊标识。默认巡检间隔为 24 小时，并发请求数限制为 5 个，避免对目标服务器造成压力。用户也可在配置文件中完全禁用该功能。

**问：如何将现有浏览器书签或收藏夹数据导入本系统？**

答：项目提供了 `scripts/import-from-html.js` 工具，可将 Chrome 或 Firefox 导出的书签 HTML 文件解析并转换为内部 JSON 格式。同时支持从 CSV 文件导入，字段映射规则详见 `docs/developer/import-export.md`。导入后建议运行校验脚本检查链接有效性。
