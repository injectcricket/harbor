# NovaLink 技术资源聚合导航

NovaLink 是一个面向开发人员、技术研究人员与运维工程师的轻量级外链资源导航系统，专用于聚合、分类与快速检索互联网中的技术文档、赛事数据、实时比分与行业资讯类外部资源。该项目不存储任何用户数据与业务内容，仅作为结构化链接入口，帮助技术团队在信息过载的环境中高效定位高价值外部站点。

项目目标用户包括个人开发者、中小型技术团队、数据采集工程师以及运维管理人员。NovaLink 通过目录化组织、标签过滤与简洁的 CLI 管理界面，将分散的外部资源整合为可维护的导航树，显著降低资源查找与团队共享的时间成本。

## 功能概览

- **多级资源目录管理**：支持自定义分类与子分类，可将外部链接按领域、用途或数据来源进行层级组织，便于团队统一入口。

- **链接状态健康检查**：内置基于 HTTP 状态码的定时检测机制，自动标记失效或响应超时的外部链接，并生成异常报告。

- **标签化快速筛选**：每条资源可绑定多个标签，支持按标签组合进行即时过滤，适用于赛事数据、开发文档与运维监控等混合场景。

- **只读只写分离架构**：导航数据的编辑与前端展示分离，编辑操作通过 CLI 工具完成，前端仅提供静态只读视图，提升安全性与稳定性。

- **Markdown 格式导出**：支持将当前资源列表导出为标准 Markdown 文档，便于集成至项目 README、Wiki 或内部知识库。

- **资源变更审计日志**：记录每条外部链接的增删改操作时间与操作者，满足团队协作与变更追溯需求。

- **低依赖部署方案**：仅依赖 Python 3.9 及以上版本与标准库，无需额外数据库或中间件，适合资源受限环境。

## 应用场景

- **技术团队内部知识库导航**：开发团队可将日常使用的 API 文档、监控面板、日志查询工具与 CI/CD 系统入口统一托管于 NovaLink，新成员入职时仅需获取一份导航配置即可快速上手。

- **赛事数据聚合入口**：数据分析师或运维人员需要同时关注多个赛事比分、赛程与排名页面时，可通过 NovaLink 将相关外链按赛事类别分组，配合健康检查功能及时发现页面访问异常。

- **运维监控仪表板补充**：运维工程师可将各环境的状态页、告警查询地址与性能图表链接集中管理，结合标签筛选实现开发环境、测试环境与生产环境的快速切换。

- **开源项目外部依赖索引**：开源维护者可在项目仓库中嵌入 NovaLink 导出的资源列表，清晰标注上游数据源、参考文档与社区论坛，降低贡献者的信息查找门槛。

## 快速开始

以下步骤适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/novalink-dev/novalink.git
cd novalink

# 安装依赖（使用 venv 隔离环境）
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt

# 初始化默认配置与目录结构
python bin/init_novalink.py --config config/default.yaml

# 启动本地预览服务（开发模式）
python bin/serve.py --host 127.0.0.1 --port 8080
```

访问 <code>http://127.0.0.1:8080</code> 即可查看导航首页。生产环境部署建议使用 Nginx 或 Caddy 代理静态文件，具体参考 `docs/deployment.md`。

## 常见问题

**问：NovaLink 是否存储外部站点的内容或用户数据？**

答：否。NovaLink 仅存储外部链接的 URL、分类标签与描述元数据，不抓取、缓存或代理任何外部站点的内容。所有用户访问行为由浏览器直接与外部站点交互，系统不记录任何个人身份信息或访问日志。

**问：健康检查模块误报外部站点不可用怎么办？**

答：健康检查模块默认使用 HTTP 状态码与超时阈值判断可用性。若目标站点存在反爬机制或响应较慢，可在 `config/default.yaml` 中调整 `health_check.timeout` 参数（单位秒）或 `health_check.retry` 重试次数。同时支持手动将特定链接加入白名单，跳过自动检查。

**问：如何批量导入已有的外链列表？**

答：NovaLink 提供 `bin/import.py` 工具，支持 CSV 与 JSON 格式的批量导入。导入前请参考 `data/links.json` 的结构预先准备数据，或使用 `bin/export.py` 导出模板后填充。导入操作会写入审计日志，便于后续追溯。
