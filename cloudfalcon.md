# Resource Navigator

Resource Navigator 是一个面向技术开发者与开源项目维护者的外链资源聚合与导航系统。该项目定位于解决开发者在日常工作中检索高质量技术文档、数据接口、官方公告页等外部资源时效率低下、入口分散的问题，通过结构化的资源分类与本地化的快速检索机制，将零散的外部链接转化为可复用、可共享、可版本化的资源清单。目标用户包括个人开发者、中小型团队的技术负责人以及开源项目的文档维护者。

项目本身不存储任何第三方数据内容，仅提供外部资源的元信息组织与展示框架。通过标准化的配置格式，用户可以自行维护资源列表，并利用内置的静态站点生成功能将资源目录发布为内部知识库或公开文档站点。项目核心价值在于将人工收藏夹的随意性转化为工程化的资源管理能力，同时保留完全的控制权与定制空间。

## 功能概览

- **结构化资源目录** 支持将外部链接按主题、用途或来源进行多级分类，每个资源条目可附带标题、描述、标签及维护状态，便于团队内部达成资源认知共识。

- **本地化检索与过滤** 内置基于关键词和标签的轻量级检索引擎，无需依赖外部搜索服务即可在数百条资源中快速定位目标链接，检索响应时间控制在毫秒级。

- **资源可用性监控** 周期性对已收录的外部链接进行可达性探测，自动标记失效或响应超时的链接，并生成异常报告，帮助维护者及时清理或更新资源。

- **批量导入与导出** 支持从 CSV、JSON 及标准浏览器书签 HTML 格式批量导入现有收藏数据，同时支持将资源列表导出为多种格式，便于迁移或备份。

- **版本化资源变更追踪** 每一条资源的增删改操作均记录变更日志，支持回溯历史版本，便于团队协作场景下追溯资源来源与变更原因。

- **静态站点一键生成** 内置模板引擎，可将当前资源目录渲染为纯静态 HTML 文档，无需数据库或后端服务即可部署至任何 Web 服务器或对象存储平台。

- **自定义元数据扩展** 允许用户为每个资源条目定义自定义字段，例如关联的项目编号、内部审核状态、替代镜像地址等，满足不同组织的个性化管理需求。

## 应用场景

- **技术团队内部文档统一入口** 开发团队可将常用的 API 文档、设计规范、CI/CD 配置参考、日志查询平台等链接统一收录至 Resource Navigator，作为团队 Onboarding 文档的标准组成部分，新成员加入时通过一份资源清单即可快速了解团队依赖的外部服务与工具链。

- **开源项目外部依赖资源索引** 开源项目维护者可在项目仓库中维护一份 resources.yml 文件，列出项目依赖的第三方库主页、数据源官网、协议文本、社区论坛等外部链接，Resource Navigator 可将其渲染为美观的文档页面，与 README 形成互补，避免 README 因链接过多而臃肿。

- **数据采集项目的源站点管理** 涉及数据爬取或 API 调用的项目，可通过该工具集中管理所有数据源 URL、备用域名、接口状态监控入口，当某个源站点变更域名或下线时，可快速在资源清单中更新并通知团队成员，减少因源站变更导致的数据任务失败。

## 快速开始

以下步骤指导您在本地环境中完成 Resource Navigator 的克隆、依赖安装与服务启动。

```bash
# 克隆项目仓库至本地
git clone https://github.com/resource-navigator/core.git resource-navigator

# 进入项目根目录
cd resource-navigator

# 安装项目依赖（使用 npm 或 yarn）
npm install

# 启动开发服务器，默认监听 3000 端口
npm run dev
```

启动成功后，访问控制台输出的本地地址即可进入资源管理界面。首次启动时会自动生成示例资源数据，您可通过管理面板进行增删改操作，或直接编辑项目根目录下的 data/resources.json 文件。

## 常见问题

**问：资源可达性监控误报率较高，部分网站因反爬机制返回 403 状态码被标记为失效，如何解决？**

答：默认监控策略仅检测 HTTP 状态码是否为 2xx 或 3xx。对于存在反爬机制的网站，您可以在 config/custom.yaml 中为特定域名配置白名单策略，或调整监控请求头中的 User-Agent 字段。更灵活的方式是自定义监控回调函数，根据响应内容而非仅依赖状态码进行判断。具体配置方法请参考管理员手册中的监控调优章节。

**问：如何将现有的浏览器书签大量导入到 Resource Navigator 中？**

答：项目支持从标准 HTML 书签导出文件（所有主流浏览器均支持此导出格式）进行批量导入。您可以在管理界面的导入功能中选择该文件，系统会自动解析书签文件夹结构并映射为资源分类标签。若您的书签数量超过 500 条，建议通过命令行工具执行导入以避免浏览器超时，具体命令为 `npm run import:bookmarks -- --file=bookmarks.html`。

**问：静态站点生成后，部分外部链接在部署到内网服务器后无法访问，是什么原因？**

答：静态站点本身不改变任何链接内容，若内网环境无法访问特定外部域名，属于网络策略限制。解决方案有两种：其一，在生成静态站点前，通过资源管理界面将内网可访问的镜像地址或代理地址更新到对应资源条目中；其二，在部署静态站点的 Web 服务器上配置反向代理，将特定外部域名代理为内网可路由的地址。详细操作请参考管理员手册中的离线部署章节。
