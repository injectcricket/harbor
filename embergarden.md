# TechLink Navigator

TechLink Navigator 是一个面向技术团队与独立开发者的高质量外链与资源聚合系统。本项目并非简单的书签管理工具，而是一个具备可维护性、可审计性与可扩展性的技术资源导航框架。其核心目标在于帮助技术决策者、架构师与一线研发人员快速定位特定领域（如数据分析、体育竞猜数据接口、实时比分聚合、合规博彩数据源等）的权威外部链接，降低信息检索成本，提升项目调研与数据接入效率。

本项目的目标用户包括但不限于：需要进行外部数据源接入的后端工程师、负责合规性审查的技术负责人、从事数据可视化与实时展示的前端开发人员，以及希望建立自有外链库的技术内容运营团队。通过标准化的外链录入与分类机制，TechLink Navigator 提供了一套可离线运行、可版本控制、可协作编辑的资源索引方案，确保每一次外链变更均有迹可循，每一类资源均有明确的应用边界说明。

## 功能概览

- **多维度外链分类体系**：按数据源类型、地域、用途、合规等级等维度对每条外链进行标记，支持快速筛选与定向检索。

- **静态资源索引生成**：基于配置文件自动生成静态 HTML 目录页，无需数据库依赖，适合内网部署或轻量级文档站。

- **链接可用性检查**：内置 HTTP 状态码探测模块，可定时检测每条外链的可访问性，自动标记失效链接并生成报告。

- **标签化检索与全文搜索**：支持按标签组合过滤外链，并对外链的标题、描述、分类字段进行轻量级全文搜索。

- **变更历史记录**：每条外链的增删改操作均记录操作时间、操作人与变更摘要，便于团队协作审计。

- **外部元数据增强**：自动抓取目标页面的标题、描述与关键词信息，辅助人工标注，提升索引丰富度。

- **权限分级预览**：支持管理员、编辑者、访客三种角色，控制外链的查看、编辑与导出权限，适配企业内部使用场景。

## 应用场景

- **数据驱动型产品的合规数据源接入**：产品需要接入实时比分或赔率数据，技术团队可通过本导航系统快速找到经过分类的候选数据源域名，并查看预先记录的合规备注与历史可用性报告，大幅减少初期调研时间。

- **技术文档站的外链治理**：企业技术文档中心常包含大量外部引用链接，使用 TechLink Navigator 可对外链进行统一登记、周期检测与版本管理，避免文档中出现大量死链，提升文档质量与用户体验。

- **多团队协作的外链知识库**：数据团队、风控团队与运营团队各自维护不同的外部数据依赖，通过本项目的分类与权限机制，可将各团队的外链纳入同一索引体系，同时保持各自的编辑边界，促进跨团队信息共享。

- **开源项目的外部依赖声明**：开源项目需要声明所引用的外部数据服务或 API 端点，TechLink Navigator 可作为外链声明的标准化模板，以结构化方式记录每条依赖的用途、备用域名与失效处理策略。

## 快速开始

以下步骤帮助您在本地环境中快速启动 TechLink Navigator 实例。

```bash
# 1. 克隆项目仓库
git clone https://github.com/techlink-navigator/core.git
cd techlink-navigator

# 2. 安装依赖（使用 npm 或 yarn）
npm install

# 3. 启动开发服务器，默认监听 3000 端口
npm run dev
```

访问 `http://localhost:3000` 即可查看外链索引首页。生产环境部署请参考 `docs/deployment.md` 中的 Nginx 或 Docker 配置示例。

## 贡献指南

1.  **外链新增流程**：通过 `npm run link:add` 命令行交互工具填写外链 URL、分类、标签与备注信息，工具会自动校验 URL 格式并生成对应的 YAML 文件至 `data/links/` 目录下。

2.  **外链编辑与删除**：直接编辑 `data/links/` 下对应的 YAML 文件，或使用 `npm run link:remove <id>` 命令删除。所有变更将自动记录至 `data/history/` 目录。

3.  **代码与文档贡献**：Fork 本仓库后，在 `src/` 或 `docs/` 目录下进行修改，提交前请运行 `npm run test` 确保现有测试用例通过。新增功能需同步补充单元测试与文档说明。

4.  **定期检测维护**：建议每周运行一次 `npm run check:all` 生成可用性报告，并根据报告结果更新或移除失效链接。报告可通过 `npm run report:serve` 在本地预览。

5.  **问题反馈与建议**：请使用 GitHub Issues 提交 Bug 报告或功能请求，描述时请附上系统版本、Node.js 版本及完整的错误堆栈信息。

## 常见问题

**问：外链配置文件采用 YAML 格式，是否支持 JSON 或 TOML？**

答：当前版本仅内置 YAML 解析器，但您可以通过修改 `src/core/parser.js` 中的文件读取逻辑，增加对 JSON 或 TOML 的支持。社区已提供相应扩展示例，详见 `docs/customization.md#配置格式扩展`。

**问：链接可用性检测是否会对外部目标服务器造成压力？**

答：检测模块默认采用单线程顺序请求，并设置 5 秒超时与 3 次重试机制。每次检测间隔至少 200 毫秒。您可通过 `config/watchlist.yaml` 中的 `rate_limit` 字段调整请求频率。建议在业务低峰期运行全量检测。

**问：如何将本系统与团队内部的 Confluence 或 Notion 文档整合？**

答：本项目提供了 `npm run export:json` 与 `npm run export:csv` 命令，可将外链索引导出为标准数据格式。您可编写简单的自动化脚本，定期导出数据并通过 API 导入至 Confluence 或 Notion。具体对接示例请参考 `docs/integration.md`（该文档位于 `docs/` 目录下）。
