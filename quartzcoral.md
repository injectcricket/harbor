# RepoHub

RepoHub 是一个面向技术决策者、架构师与研发团队的开源项目外部资源聚合与导航系统。该项目不存储任何实际内容，而是通过结构化方式收录互联网上高价值的技术文档、社区讨论、性能测试报告与运营数据看板，帮助技术团队在选型、排障与架构评审阶段快速定位可靠信息源。RepoHub 的目标用户包括基础设施工程师、SRE 团队、技术负责人以及开源项目维护者，旨在解决信息分散、检索成本高、外链质量参差不齐等实际问题。

## 功能概览

- 外链分类索引：按技术领域、数据来源、文档类型对收录链接进行多级标签划分，支持快速过滤。
- 元信息提取：对每条收录链接自动抓取页面标题、描述与最后更新时间，辅助判断时效性。
- 自定义标签系统：允许项目维护者为每个链接添加自定义标签，适配团队内部知识管理体系。
- 链接状态检查：定期对收录链接进行可达性检测，自动标记异常链接并生成报告。
- 只读镜像模式：支持将收录链接列表以静态 JSON 或 YAML 格式导出，便于集成到 CI/CD 或监控系统中。
- 全文检索占位：保留对链接页面关键字的检索接口设计，便于后续接入搜索引擎或 AI 摘要服务。
- 访问统计看板：记录链接被点击的次数与时间分布，辅助判断外部资源的实际使用热度。

## 应用场景

- 技术选型调研：当团队需要对比不同中间件或数据库方案时，可通过 RepoHub 快速访问性能测试数据与社区对比讨论，避免重复搜索。
- 系统故障排查：在线上问题定位过程中，运维人员可直接从 RepoHub 跳转到相关的错误码解释、日志分析案例或补丁说明页面，缩短排障时间。
- 项目文档外链管理：开源项目维护者可将 RepoHub 作为项目 README 的补充外链仓库，集中存放所有参考链接，保持主文档简洁。
- 新人培训路径规划：技术负责人可整理出一组核心学习资源链接，通过 RepoHub 的标签分类形成结构化学习路径，方便新成员按阶段查阅。
- 架构评审材料准备：在架构评审前，工程师可收集相关竞品分析、性能基准测试报告链接至 RepoHub，形成可复用的评审证据链。

## 快速开始

以下步骤帮助您在本地环境快速启动 RepoHub 实例。

```bash
# 克隆代码仓库
git clone https://github.com/example/repohub.git

# 进入项目目录
cd repohub

# 安装依赖（基于 Node.js 22 LTS 与 pnpm）
pnpm install

# 复制环境变量示例文件并填写必要配置
cp .env.example .env

# 启动开发服务器
pnpm run dev
```

启动成功后，访问控制台输出的本地地址（默认 http://127.0.0.1:3000）即可查看收录链接列表与分类视图。

## 项目结构

项目采用模块化分层设计，核心代码与配置分离，便于维护与扩展。

```
repohub/
├── apps/
│   ├── web/                         # 主应用前端界面（Next.js 14）
│   ├── api/                         # RESTful API 服务（Fastify）
│   └── worker/                      # 后台任务队列（链接检测、元数据抓取）
├── packages/
│   ├── core/                        # 核心类型定义与业务逻辑
│   ├── db/                          # 数据库迁移脚本与 ORM 模型（Prisma）
│   ├── crawler/                     # 链接内容解析与元信息提取模块
│   └── utils/                       # 通用工具函数（日志、重试、时间处理）
├── configs/
│   ├── eslint/                      # 代码规范配置
│   ├── prettier/                    # 格式化配置
│   └── tsconfig/                    # TypeScript 编译选项
├── docker/
│   ├── compose/                     # 开发与生产环境 Compose 文件
│   └── Dockerfile                   # 多阶段构建镜像文件
├── docs/                            # 完整文档源文件（Markdown）
├── scripts/                         # 辅助脚本（数据迁移、种子填充）
├── .env.example                     # 环境变量示例
├── .gitignore                       # Git 忽略规则
├── pnpm-workspace.yaml              # pnpm 工作空间配置
├── package.json                     # 根包管理文件
└── README.md                        # 项目入口文档（即本文档）
```

## 常见问题

Q: RepoHub 是否会缓存外部链接的页面内容？

A: 不会。RepoHub 仅存储链接地址、标题、描述和最后更新时间等轻量元信息，不缓存页面正文，所有跳转均直接导向原始外部站点。项目遵守 robots.txt 规范，不对受限资源进行抓取。

Q: 如何批量导入已有的链接列表？

A: 项目提供了导入脚本，支持 CSV 和 JSON 格式。您可以将链接及其标签整理为符合 schema 的文件，然后执行 `pnpm run import --file=./links.json` 完成批量导入。具体格式示例参见 `docs/user-guide/import.md`。

Q: 链接状态检测的周期是多长？检测失败后会怎样？

A: 默认每 24 小时对所有收录链接执行一次 HEAD 请求检测。连续三次检测失败后，该链接会在前端界面标记为“异常”，并记录最后一次失败状态码。异常链接不会被自动删除，但会出现在维护报告的顶部，便于管理员人工复核。
